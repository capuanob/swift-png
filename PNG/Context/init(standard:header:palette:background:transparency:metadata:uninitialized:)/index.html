Initializer

init(standard:header:palette:background:transparency:metadata:uninitialized:)

Creates a fresh decoding context.

Declaration

init?(standard:Standard, header:Header, palette:Palette?, background:Background?, transparency:Transparency?, metadata:Metadata, uninitialized:Bool)

Parameters

standard

The PNG standard of the image being decoded. This should be ios if the image began with a CgBI chunk, and common otherwise.

header

The header of the image being decoded. This is expected to have been parsed from a previously-encountered IHDR chunk.

palette

The palette of the image being decoded, if present. If not nil, this is expected to have been parsed from a previously-encountered PLTE chunk.

background

The background descriptor of the image being decoded, if present. If not nil, this is expected to have been parsed from a previously-encountered bKGD chunk.

transparency

The transparency descriptor of the image being decoded, if present. If not nil, this is expected to have been parsed from a previously-encountered tRNS chunk.

metadata

A metadata instance. It is expected to contain metadata from all previously-encountered ancillary chunks, with the exception of bKGD and tRNS.

uninitialized

Specifies if the image storage should be initialized. If false, the storage buffer will be initialized to all zeros. This can be safely set to true if there is no need to access the image while it is in a partially-decoded state.

The default value is true.

Overview

It is expected that client applications will initialize a decoding context upon encountering the first IDAT chunk in the image.
